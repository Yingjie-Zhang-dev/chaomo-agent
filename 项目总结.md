# 潮墨智能体 - 项目开发与部署总结

## 项目概述

**潮墨智能体**是一个基于水墨艺术风格的 AI 智能助手 Web 应用，用户可通过浏览器访问，体验水墨知识问答、作品评价和创意图像生成三大功能。

- **在线访问**：http://1b693889.r39.cpolar.top/agent/
- **GitHub**：https://github.com/Yingjie-Zhang-dev/chaomo-agent
- **码云**：https://gitee.com/jackies-melody-maker/chaomo-agent

### 核心技术栈

| 类别 | 技术 |
|------|------|
| 前端 | 原生 HTML/CSS/JavaScript（无框架） |
| Markdown 渲染 | marked.min.js |

## 功能模块

### 1. 水墨知识站（文生文）
- 基于 Dify 对话流应用
- 连接水墨艺术知识库
- 支持流式输出（SSE）
- 用途：回答水墨画技法、历史、鉴赏等问题

### 2. 作品交流馆（图生文）
- 上传水墨作品图片
- Dify LLM 进行视觉分析
- 返回专业点评和鉴赏建议
- 同步输出模式
- 用户上传的图片通过 server.js 保存

### 3. 文创实训坊（图生图）
- 上传参考图片 + 文字描述
- Dify 工作流调用 Seedream 工具
- 生成水墨风格创意图像
- 同步输出模式

## Dify 后端配置

### 工作流导入
在 Dify 平台导入以下 YAML 文件：
| 文件 | 功能 |
|------|------|
| `文生文.yml` | 水墨知识问答 |
| `图生文.yml` | 作品评价 |
| `图生图.yml` | 图像生成 |

### Nginx 代理配置要点
Dify 部署时，Nginx 路径重写是关键：

```nginx
# API 代理
location /agent/v1 {
    proxy_pass http://api:5001/v1;
    include proxy.conf;
}

# 文件上传代理 - 转发到本地 server.js
location /agent/upload {
    proxy_pass http://localhost:3001/upload;
    include proxy.conf;
}

location /agent/uploads {
    proxy_pass http://localhost:3001/uploads;
    include proxy.conf;
}

# 文件服务代理
location /agent/files {
    proxy_pass http://api:5001/files;
    include proxy.conf;
}
```

## 部署流程

### 开发到上线流程

1. **本地开发**：修改 `index.html`、`server.js` 等文件
2. **提交 Git**：完成阶段性开发后 commit
3. **同步到 Dify**：将修改后的文件复制到 nginx 目录
4. **重启服务**：让改动生效

### 同步文件到 Dify 服务器

```bash

   "C:/Users/zhang/Downloads/dify-main/docker/volumes/certbot/www/index.html"
```

## 文件服务器

`server.js` 是本地文件服务器，负责保存用户上传的图片：

- **端口**：3001
- **上传目录**：`uploads/`
- **启动命令**：`node server.js`

**公网访问**：通过 nginx 代理（cpolar 免费版仅暴露 80 端口）：
- 公网请求 `/agent/upload` → nginx → `localhost:3001/upload`
- 公网请求 `/agent/uploads/xxx` → nginx → `localhost:3001/uploads/xxx`

## 常见问题

### 1. 提交者显示错误
GitHub 根据邮箱关联提交者，确保使用 GitHub 账号对应的邮箱。

### 2. 部署后访问 404
检查 Nginx 的 `proxy_pass` 配置是否正确添加路径后缀。

### 3. 文件上传失败
- 确认 server.js 是否启动：`node server.js`
- 确认 nginx 配置已添加 `/agent/upload` 和 `/agent/uploads` 代理

### 4. cpolar 免费版限制
- 仅暴露 80/443 端口
- 文件上传通过 nginx 代理到 server.js 解决，无需额外暴露端口
