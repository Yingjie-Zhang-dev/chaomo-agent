# 部署流程

## 本地开发与公网访问

### 工作流程

1. **本地开发**：修改 `index.html`、`server.js` 等文件
2. **提交 Git**：完成阶段性开发后 commit
3. **同步到 Dify**：将修改后的文件复制到 nginx 目录
4. **重启服务**：让改动生效

---


### 文件服务器部署

本地文件服务器 `server.js` 保存用户上传的图片：

- **启动命令**：`node server.js`
- **端口**：3001
- **上传目录**：`c:/Users/zhang/Desktop/潮墨智能体/uploads/`

如需公网用户也能上传图片并保存，需要：
1. 在 Dify 服务器上部署 server.js
2. 通过 cpolar 暴露 3001 端口

### 关键提醒

1. 每次修改 index.html 后都要复制到 nginx 目录并重启
2. 文件服务器需要手动启动，或设置开机自启
3. Dify 工作流配置在 Dify 服务器上，不在这个仓库
